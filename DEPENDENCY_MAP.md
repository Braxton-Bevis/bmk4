# KisakCOD Windows/DirectX/x86 → iOS/POSIX Dependency Map

This document is the canonical dependency map for porting KisakCOD (a Win32/x86-32/D3D9 C++ reimplementation of Call of Duty 4) to **arm64-apple-ios**. It inventories every Windows, DirectX, and x86-specific dependency found in the source tree, what the engine uses it for, where it lives, and the concrete iOS/POSIX/Metal replacement for each. It was generated on **2026-07-11** from an **11-agent source scan** (10 subsystem-scan agents covering all of `src/`, `deps/`, and `milesEq/`, plus a completeness critic) cross-checked against the **CI compile census run on the real iOS SDK** (`ios-compile-probe.yml`). Approximate call-site counts come from the scan; effort ratings are per API family, not per subsystem. Use this together with `PORT_JOURNAL.md` when sequencing the port.

## 14. Gaps found by the completeness critic

Items the 10 per-subsystem scans missed, found in the dedicated gaps pass:

1. **setjmp/longjmp error-unwinding architecture** — the engine's core error recovery (`Com_Error`, GSC script "exceptions", fastfile load errors, render backend) was never mentioned by any subsystem agent. `jmp_buf g_com_error[THREAD_CONTEXT_COUNT]` at `src/universal/q_shared.cpp:153`; `jmp_buf g_script_error[33]` at `src/script/scr_vm.cpp:69` with longjmps at `scr_vm.cpp:3960,4440`, `scr_evaluate.cpp:1384,1475`; also `qcommon/common.cpp:695,1072,2135,2374`, `db_registry.cpp:2326`, `sv_main.cpp:499,505` (jmp_buf smuggled through `Sys_GetValue(2)` as raw `int*`), `rb_backend.cpp:2818`, `r_workercmds.cpp:496`. `jmp_buf` is 64 B on MSVC x86 but ~192 B on arm64-apple — ABI-sized storage corrupts memory; `longjmp` across C++/Obj-C++ frames skips destructors under clang. Must be a first-class port item.
2. **Tracy profiler is a live third-party dependency** compiled into all three targets — absent from every agent's list. `scripts/extern/tracy.cmake` FetchContent (github.com/wolfpld/tracy v0.12.2) at configure time; included by `scripts/mp/CMakeLists.txt:71`, `scripts/sp/CMakeLists.txt:64`, `scripts/dedi/CMakeLists.txt:66`; hooks at `src/universal/profile.h:1390-1391`, `src/qcommon/common.cpp:2128,2169`, `src/qcommon/threads.cpp:204`. Debug iOS builds pull a network fetch at configure time and set an `MSVC_RUNTIME_LIBRARY` property meaningless on Apple toolchains.
3. **Generated header `src/buildnumber.h`** — produced by a pre-build script, not in the tree; consumers: `src/qcommon/common.cpp:25`, `src/client/cl_console.cpp:10`, `src/ui_mp/ui_main_mp.cpp:12`, `src/game/savememory.cpp:7`, `src/buildnumber.cpp:1`. `scripts/ios/CMakeLists.txt` neither includes `pre_build.cmake` nor depends on `update_build_number`, so these TUs fail with a missing-header error unrelated to Windows APIs. Check in a stub or wire the target.
4. **Backslash OS-path construction and Windows filesystem semantics** baked into "portable" layers — agents flagged the file APIs but not the path *strings*: `db_registry.cpp:598,2733` (fastfile open paths `"%s\\zone\\%s\\%s.ff"`), `com_sndalias.cpp:356`, `win_common.cpp:177,204,218`, `assertive.cpp:449,465`, `q_shared.h:98` (`#define PATH_SEP '\\'`), and `FS_FilenameCompare` (`com_files.cpp:723-744`) folding case and treating `\`/`:` as `/` while iOS APFS is case-sensitive. A distinct cross-cutting task from swapping I/O calls.
5. **Display-settings/GDI cluster in src/universal** — `FixWindowsDesktop()` (`assertive.cpp:619-640`) resets desktop display mode (`ChangeDisplaySettingsA`), gamma ramp (`SetDeviceGammaRamp`), and window styles (`EnumThreadWindows`/`SetWindowLongA`) — an entire user32/gdi32 family appearing nowhere else, compiled into every target including dedi.
6. **The `WIN32` macro is injected by CMake, not the compiler** — `scripts/pre_build.cmake:22` (`target_compile_definitions(... PUBLIC WIN32 _CONSOLE _MBCS)` unconditionally, on any host OS). All `#ifdef WIN32` gates (Steam `#error`s etc.) are driven by the build system; reusing `pre_build.cmake` for an Apple target would re-enable every Win32 path. Latent script defects: `:17-19` missing PUBLIC/PRIVATE keyword for `KISAK_EXTENDED`; `scripts/platform/win32/platform.cmake:14-15` references undefined `MSVC_WARNING_DISABLES`; `:18` passes deliberately bogus flag `/intentionallbreakthisshit`; `:78` `/machine:x86`; `:106` static CRT; `:49-51` `DXSDK_DIR` env-var dependency for D3DX.
7. **Hardcoded absolute Windows paths in build scripts** — `milesEq/build.bat` (VS2022 vcvarsall, `C:\MilesWinBaby` SDK includes/libs, compiles `C:\MilesWinBaby\src\sdk\ribdll.c`), `milesEq/verify.bat`, `scripts/mksln.bat` (pins `-G "Visual Studio 17 2022" -A Win32`). The pervasive `c:\trees\cod3\...` strings in src/ are assert/log literals only — no runtime file access.
8. **`_putenv` MSVC-CRT semantics bug in `CL_Setenv_f`** — `src/client_mp/cl_main_mp.cpp:646` (getenv) and `:670` (`_putenv(buffer)` on a 1028-byte stack array). `_putenv` is MSVC-only (link error on iOS); renaming to POSIX `putenv` creates a use-after-scope (POSIX stores the caller's pointer). Use `setenv()`.
9. **No headless target exists** — `scripts/dedi/CMakeLists.txt:25,35,53` compiles the full D3D9 renderer (`${GFX_D3D}`), Bink (`${BINKLIB}`), and the whole `src/win32` layer (`${WIN32}`), and includes `pre_build.cmake` (line 62) linking d3d9/ddraw/mss32/binkw32/steam_api like the client. The "port the dedicated server first, it has no renderer" shortcut does not exist here; any minimum-viable iOS target brings the renderer and platform layer immediately.

## 15. Known blockers with no direct replacement

These cannot be ported — only replaced, redesigned, or deleted:

1. **Miles Sound System** (`mss32.dll` + 8 `.asi`/`.flt` plugin codecs) — binary-only, 32-bit x86 PE, **no ARM64/iOS build exists** in the repo or from the era. The 141 AIL_* references (58 distinct functions, concentrated in `src/sound/snd_driver.cpp` and `snd_mss.cpp`) must be re-fronted by an AVAudioEngine/CoreAudio shim; the runtime plugin-DLL mechanism is impossible on iOS and its DSP/codecs must be statically compiled in.
2. **Bink video** (`binkw32.dll`) — binary-only 32-bit x86 PE. Requires either a RAD Bink 2 iOS SDK license (arm64 libraries exist commercially) or full replacement of the cinematic path (`src/gfx_d3d/r_cinematic.cpp`, `deps/binklib/binktextures.cpp`) with AVFoundation plus re-encoding all `.bik` assets.
3. **Steam API** (`steam_api.dll`, Steamworks auth in `src/win32/win_steam.cpp` + hard `#error` gates in 5 client/server files) — **no Steamworks runtime exists for iOS**. Auth-ticket/GUID flow must be stubbed (LAN path already bypasses) or replaced with a custom token service; SteamID64-as-player-GUID needs an alternate identity.
4. **IPX networking** (`wsipx.h`, `NSPROTO_IPX` in `src/win32/win_net.cpp`) — dead legacy transport with no modern equivalent on any platform. **Delete.**
5. **249 x86-32 structure-layout static_asserts + fastfile serialization** — the IWff0100 zone format and the GSC VM value unions bake 4-byte pointers into on-disk and in-memory layouts (`src/database/db_load.cpp`, `scr_variable.h`, `scr_debugger.h`, `directsound.h:50`, `snd_public.h:110`, savegames in `g_save.cpp`/`savememory.cpp`). There is no mechanical fix: each requires LP64 widening + assert rebase, a load-time translation layer, or 32-bit arena-offset handles. This plus the Metal backend is the critical path of the port.
6. **73 inline `__asm` blocks** — none compile under arm64 clang. The two live ones (`SnapFloatToInt` x87 in `qcommon.h:1587-1599`, EBP stack walk in `assertive.cpp:547-549`) have exact arm64 replacements (`vcvtns_s32_f32`, `backtrace()`); the remaining ~70 are `#if 0`/commented/config-gated decompile remnants (PowerPC `fsel`, cpuid, PENTIUM rdtsc, ARM32 speex) that must stay disabled or be deleted.
